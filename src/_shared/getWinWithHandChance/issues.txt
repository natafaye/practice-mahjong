1. Probability Summation Error (Non-Disjoint Events)
The function sums the probabilities of all possible joker combinations. However, these
combinations are not mutually exclusive. For example, if you need a 1-Dot and have one Joker available in the wall:
  * Combo A: Draw the 1-Dot naturally.
  * Combo B: Draw the Joker to substitute for the 1-Dot.

Drawing both the 1-Dot and the Joker satisfies both combinations. Summing their probabilities double-counts these overlapping        
outcomes, which overestimates the chance and can lead to a drawingChance greater than 1.

2. Conservative "Exactly" Logic
The multivariate hypergeometric formula implementation calculates the probability of drawing exactly the number of tiles specified in
a combo and zero extra of any tiles listed in totalAvailable.

In Mahjong, drawing more than the required number of a needed tile is still a win. By excluding outcomes where extra winning tiles   
are drawn, the function significantly underestimates the probability of each individual combination.
